# Regra de Desenvolvimento

Antes de implementar qualquer feature:
1. Explicar abordagem
2. Explicar arquivos envolvidos
3. Explicar fluxo de dados
4. Só então implementar

---

# Regra Principal

Priorizar:
- simplicidade
- acessibilidade
- conforto visual
- clareza
- experiência humana

Nunca priorizar estética acima da usabilidade.
