# Diretrizes de UI/UX — Aplicativo de Fibromialgia

## Objetivo da Interface

A interface deve transmitir acolhimento, leveza, simplicidade e conforto emocional.

O aplicativo será utilizado por pessoas com:
- dores crônicas
- fadiga mental
- sensibilidade emocional
- ansiedade
- baixa energia cognitiva

Portanto, toda experiência visual deve reduzir fricção, estresse e sobrecarga mental.

O design deve parecer humano, funcional e emocionalmente confortável.

A prioridade absoluta é:
- clareza
- conforto visual
- facilidade de uso
- acessibilidade
- experiência acolhedora

A estética nunca deve ser mais importante do que a usabilidade.

---

# Filosofia Visual

## O sistema NÃO deve possuir aparência genérica de IA

Evitar:
- aparência de template SaaS
- dashboards futuristas
- UI “tech”
- excesso de gradientes
- excesso de glassmorphism
- interfaces exageradamente modernas
- componentes decorativos sem propósito
- excesso de widgets
- layouts artificiais
- visual “Dribbble concept”
- aparência de startup de IA

O aplicativo deve parecer:
- humano
- natural
- discreto
- confiável
- acolhedor
- simples
- emocionalmente confortável

---

# Sensação Emocional do Produto

A experiência deve transmitir:

- calma
- leveza
- acolhimento
- segurança emocional
- simplicidade
- clareza
- confiança
- cuidado

O usuário nunca deve se sentir:
- pressionado
- perdido
- sobrecarregado
- cansado visualmente

---

# UX First

A experiência do usuário é mais importante do que aparência visual.

Toda decisão de design deve priorizar:
1. clareza
2. simplicidade
3. acessibilidade
4. conforto
5. velocidade de compreensão

---

# Diretrizes de Layout

## Estrutura Geral

- Layouts simples
- Muito espaço em branco
- Navegação intuitiva
- Poucos elementos por tela
- Hierarquia visual clara
- Separação visual suave entre seções

Evitar:
- telas carregadas
- excesso de informação
- múltiplos focos visuais
- excesso de elementos simultâneos

---

# Espaçamento

Utilizar espaçamentos generosos para criar sensação de respiro.

Preferir:
- padding confortável
- margens suaves
- separação clara entre blocos

Evitar:
- componentes colados
- densidade excessiva
- telas comprimidas

---

# Componentes

## Regras Gerais

Todo componente deve possuir propósito claro.

Evitar:
- componentes puramente decorativos
- excesso de cards
- excesso de badges
- excesso de ícones
- excesso de elementos interativos

Priorizar:
- simplicidade
- legibilidade
- clareza funcional

---

# Cards

Cards devem ser utilizados apenas quando ajudarem na organização.

Evitar:
- dashboards com dezenas de cards
- cards decorativos
- profundidade exagerada
- sombras fortes

Preferir:
- cards leves
- sombras suaves
- bordas discretas
- organização limpa

---

# Bordas e Arredondamento

Utilizar arredondamento suave e natural.

Evitar:
- bordas extremamente arredondadas
- aparência infantilizada
- excesso de curvas

Preferir:
- rounded-md
- rounded-lg discretos

---

# Sombras

Sombras devem ser suaves e discretas.

Evitar:
- sombras pesadas
- glow effects
- efeitos neon
- profundidade exagerada

---

# Animações

Animações devem ser mínimas e funcionais.

Evitar:
- animações constantes
- microinterações excessivas
- movimentos chamativos
- transições longas

Priorizar:
- transições suaves
- feedback discreto
- movimento mínimo

A interface deve parecer estável e calma.

---

# Tipografia

## Objetivos

A tipografia deve:
- reduzir esforço cognitivo
- facilitar leitura rápida
- transmitir conforto

---

# Regras Tipográficas

Preferir:
- fontes limpas
- boa altura de linha
- contraste confortável
- tamanhos acessíveis

Evitar:
- fontes futuristas
- fontes decorativas
- textos pequenos
- blocos densos de texto

---

# Hierarquia Visual

A hierarquia deve ser clara imediatamente.

Usuário deve entender:
- onde está
- o que fazer
- o que é importante

sem esforço mental.

---

# Paleta de Cores

## Objetivo das cores

As cores devem transmitir:
- calma
- acolhimento
- leveza
- segurança

---

## Fundo
- #F7F8FC

## Roxo suave
- #8B7CF6

## Azul leve
- #7DB7FF

## Verde suave
- #9AD7B5

## Texto principal
- #2D3748

## Texto secundário
- #718096

---

# Regras de Cor

Evitar:
- neon
- contraste agressivo
- saturação exagerada
- excesso de cores simultâneas

Preferir:
- tons suaves
- baixa agressividade visual
- consistência cromática

---

# Acessibilidade

## Regras Obrigatórias

- contraste adequado
- navegação clara
- botões acessíveis
- textos legíveis
- tamanho confortável de toque
- foco visível
- responsividade completa

---

# Acessibilidade Emocional

Usuários podem estar:
- cansados
- ansiosos
- com dor
- mentalmente fatigados

Portanto:
- reduzir estímulos visuais
- evitar poluição visual
- evitar excesso de notificações
- evitar complexidade
- evitar pressão visual

---

# Mobile First

O sistema deve ser pensado prioritariamente para celular.

Priorizar:
- navegação simples
- toque confortável
- telas objetivas
- fluxo rápido

---

# Formulários

Formulários devem ser:
- curtos
- leves
- intuitivos
- rápidos de preencher

Preferir:
- sliders
- seletores simples
- emojis discretos
- inputs claros

Evitar:
- formulários longos
- excesso de campos
- esforço cognitivo alto

---

# Dashboard

O dashboard deve:
- mostrar apenas informações importantes
- evitar excesso de métricas
- ser simples de entender rapidamente

Priorizar:
- clareza
- leitura rápida
- organização limpa

Evitar:
- dashboards corporativos
- excesso de gráficos
- excesso de widgets

---

# Gráficos

Gráficos devem ser:
- simples
- suaves
- legíveis

Evitar:
- visual técnico
- excesso de linhas
- poluição de dados

---

# Ícones

Ícones devem:
- auxiliar entendimento
- ser discretos
- possuir significado claro

Evitar:
- excesso de ícones
- ícones decorativos
- visual exageradamente moderno

---

# Navegação

A navegação deve:
- exigir poucos passos
- ser previsível
- reduzir esforço mental

Usuário nunca deve se perguntar:
- “onde estou?”
- “como volto?”
- “o que faço agora?”

---

# Sensação Final Esperada

O aplicativo deve parecer:
- humano
- acolhedor
- confortável
- simples
- confiável

A experiência deve transmitir:
“esse aplicativo foi feito para cuidar de mim.”

Nunca:
“esse aplicativo foi feito para impressionar visualmente.”

---

# Referências de Experiência

Inspirar-se em aplicativos com:
- design calmo
- UX acolhedora
- experiência minimalista
- foco emocional

Referências:
- Headspace
- Calm
- Daylio
- Bearable

Não copiar visualmente.
Apenas utilizar como referência de sensação e experiência.

---

# Regras Técnicas para UI

- Utilizar TailwindCSS
- Componentização consistente
- Design system reutilizável
- Responsividade total
- Acessibilidade obrigatória
- Evitar CSS excessivamente complexo
- Manter consistência visual global

---

# Regra Final

Toda interface criada deve priorizar:
- conforto humano
- simplicidade
- clareza
- acessibilidade
- acolhimento emocional

Se uma decisão visual parecer “bonita”, mas aumentar esforço mental, ela deve ser evitada.
